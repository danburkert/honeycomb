What is Honeycomb?

Honeycomb is an open source storage engine plugin for MySQL that enables MySQL to store, modify, and query tables directly in HBase. With Honeycomb installed, applications can utilize the familiar and powerful relational data model and SQL interface while allowing for access to data with existing Hadoop-based tools.

Honeycomb was designed to preserve the investment in existing relational database applications and bridge the gap to the NoSQL world.

Currently, MySQL table data can be stored in HBase and an in-memory store. Support for additional data storage backends can be implemented without knowledge of MySQL internals through Honeycomb's high-level interfaces.

How It Works

Honeycomb is organized into three distinct layers: the storage engine plugin, proxy, and backend.

A high-level view of their interaction is shown in this figure:

The Honeycomb storage engine plugin is loaded by MySQL and is used to provide information from the MySQL optimizer to the storage engine proxy.

The proxy layer is a Java library responsible for communicating between MySQL and a storage engine backend.

A backend layer implements Honeycomb's interfaces to define how supported operations should be performed by the storage implementation (for example, HBase) to process the SQL statements issued to MySQL.

For further details, take a look at the general architecture of Honeycomb.

Feature Comparison

Honeycomb is meant to be a "drop in" replacement for the default MySQL storage engine, InnoDB.

Existing applications using MySQL may be transitioned to use HBase with Honeycomb on a table-by-table basis depending upon Honeycomb's supported feature set.

A brief feature comparison between Honeycomb with the HBase backend and MySQL with InnoDB is shown in the following table:
